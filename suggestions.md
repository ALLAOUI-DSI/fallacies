# Suggestions to improve Fallacy Fighter

## Existing ideas
- Add streak-based rewards (daily streak, weekly streak, and milestone badges) to increase retention.
- Introduce timed challenge mode with optional countdown pressure for advanced players.
- Add "Explain in your own words" free-text reflection after each quiz question.
- Offer adaptive question selection that increases/decreases difficulty from recent performance.
- Add per-category mastery tracking (relevance, causality, ambiguity, etc.).
- Include an onboarding mini-tutorial with two guided quiz examples for new users.
- Provide accessibility options: dyslexia-friendly font, larger default text, and reduced motion.
- Add export/import for local progress backup.
- Add optional multiplayer "debate duel" mode where players identify fallacies in the same prompt.
- Create teacher/classroom mode with assignable level packs and progress snapshots.

## New ideas
- Add "combo chains" — correctly identifying consecutive fallacies within a category earns bonus points and unlocks hidden trivia about the fallacy's origin.
- Create a "debate simulator" mode where the app presents a dialogue and users must identify and flag all fallacies in real-time as they appear.
- Add a "fallacy of the day" widget on the dashboard with a new example and mini-quiz each day to encourage daily practice.
- Introduce a spaced-repetition system that re-tests fallacies the user previously got wrong at increasing intervals.
- Add a "build your own fallacy" sandbox where users construct examples of a given fallacy and rate each other's creations.
- Create shareable "fallacy report cards" that summarize a user's strengths and weaknesses across categories and levels.
- Add audio narration mode for passages and explanations to support auditory learners.
- Introduce themed passage packs (politics, advertising, workplace, social media, science) for the extractor mode.
- Add a "speed round" variant for the extractor mode with a countdown timer and bonus points for quick correct answers.
- Create a progress heatmap calendar on the dashboard showing which days the user practiced.
- Add support for user-submitted passages and fallacy examples that can be reviewed and included in the game.
- Introduce "fallacy chains" where one passage contains multiple fallacies and users must identify all of them for full credit.
